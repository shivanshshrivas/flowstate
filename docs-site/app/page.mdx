# FlowState Developer Docs

`@shivanshshrivas/flowstate` is an npm package that gives e-commerce platforms:

- **One-click crypto checkout** — a drop-in `PayButton` component that handles wallet connection, token approval, and escrow deposit
- **Event-based escrow** — funds are locked on-chain via the `EscrowStateMachine` contract and released in stages as shipment milestones are reached
- **Shippo shipping tracking** — real-time carrier tracking drives automatic state transitions and partial payouts
- **Dashboard interfaces** — `OrderTracker` for buyers, seller dashboards for fulfillment, and admin analytics

The package runs on the **XRPL EVM Sidechain** and uses **FLUSD** (a mock RLUSD stablecoin) for payments. The backend API gateway is housed in `backend/` and provides all order orchestration, webhook dispatch, and third-party bridge integrations.

## Integration Workflow

1. Install `@shivanshshrivas/flowstate` in your storefront.
2. Create a project and API key (`POST /api/v1/auth/projects/create`).
3. Wrap your app with `FlowStateProvider` and supply contract addresses.
4. Add `PayButton` to product or cart pages.
5. Register a webhook endpoint (`POST /api/v1/webhooks/register`).
6. Process state-change events from webhooks and display progress with `OrderTracker`.

## Package Exports

The package provides these primary exports:

| Export                  | Kind      | Description                                                      |
| ----------------------- | --------- | ---------------------------------------------------------------- |
| `FlowStateProvider`     | Component | React context provider for project config and contract addresses |
| `PayButton`             | Component | One-click Buy with FlowState button with wallet connection       |
| `OrderTracker`          | Component | Visual timeline showing order state, payouts, and tx links       |
| `useFlowState`          | Hook      | Access resolved config and contract addresses from context       |
| `EscrowStateMachineAbi` | ABI       | Solidity ABI for the escrow contract                             |
| `MockRLUSDAbi`          | ABI       | Solidity ABI for the FLUSD ERC-20 token                          |
| `OrderState`            | Enum      | `INITIATED` through `FINALIZED` plus `DISPUTED`                  |
| `DisputeStatus`         | Enum      | `OPEN`, `SELLER_RESPONDED`, `RESOLVED`, `AUTO_RESOLVED`          |

## API Groups

The backend gateway exposes eight route groups under `/api/v1`:

- **Auth** — project creation and API key rotation
- **Orders** — checkout lifecycle: create, select shipping, confirm escrow, advance, finalize
- **Shipping** — Shippo rate shopping, tracking, inbound carrier webhook
- **Sellers** — onboarding, order queues, metrics, payout history
- **Disputes** — buyer dispute creation, seller response, admin resolution
- **Platform** — analytics, seller risk views, gas cost estimates
- **Webhooks** — destination registration and delivery logs
- **Agents** — role-based AI chat routing (buyer/seller/admin)
