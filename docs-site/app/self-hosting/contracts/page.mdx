# Contracts

The XRPL EVM smart contracts live in `packages/contracts/`.

## Setup

```bash
cd packages/contracts
cp .env.example .env
# Add your deployer private key and RPC URL
npm install
```

## Commands

```bash
npm run build          # Compile contracts with Hardhat
npm test               # Run Hardhat test suite
npm run deploy:testnet # Deploy to XRPL EVM testnet
npm run seed:testnet   # Seed contracts with test data
npm run demo:testnet   # Run demo interaction script
```

## Contract Details

### FLUSD

- ERC-20 with `"Flow USD"` name, `"FLUSD"` symbol, 6 decimals
- `faucet()` — public, mints 50,000 FLUSD, 1-hour cooldown per address
- `mint(to, amount)` — owner only

### EscrowStateMachine

- `createEscrow(orderId, buyer, seller, token, amount, payoutBps[5])` — locks tokens, releases first tranche
- `advanceState(escrowId)` — moves to next state, releases corresponding payout
- `finalize(escrowId)` — releases holdback minus platform fee after grace period
- `initiateDispute(escrowId, evidenceCid)` — freezes funds, sets state to DISPUTED
- `executeResolution(escrowId, refundBps)` — called by DisputeResolver only
- Owner config: `setOperator`, `setDisputeResolver`, `setPlatformFeeWallet`, `setPlatformFeeBps` (max 10%), `setDefaultGracePeriod`, `pause`/`unpause`

### DisputeResolver

- `createDispute(escrowId, buyerEvidenceCid)` — opens dispute, freezes escrow
- `respondToDispute(disputeId, sellerEvidenceCid)` — seller submits evidence
- `resolveDispute(disputeId, outcome, refundBps)` — admin resolves
- `autoResolve(disputeId)` — anyone can call after 72h seller window expires
