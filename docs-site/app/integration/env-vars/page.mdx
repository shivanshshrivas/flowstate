# Environment Variables

This page covers variables needed by an external integrator using the `@shivanshshrivas/flowstate` package with the hosted API gateway.

## Storefront / Frontend

These are `NEXT_PUBLIC_` prefixed variables exposed to the browser:

| Variable                                   | Required      | Description                                   |
| ------------------------------------------ | ------------- | --------------------------------------------- |
| `NEXT_PUBLIC_FLOWSTATE_PROJECT_ID`         | Yes           | Your project ID from `/auth/projects/create`  |
| `NEXT_PUBLIC_FLOWSTATE_API_KEY`            | Yes           | Your API key (used by client-side components) |
| `NEXT_PUBLIC_FLOWSTATE_NETWORK`            | Yes           | `testnet` or `mainnet`                        |
| `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID`     | Recommended   | WalletConnect project ID for wallet modal     |
| `NEXT_PUBLIC_MOCK_RLUSD_ADDRESS`           | Yes (testnet) | FLUSD token contract address                  |
| `NEXT_PUBLIC_ESCROW_STATE_MACHINE_ADDRESS` | Yes           | EscrowStateMachine contract address           |
| `NEXT_PUBLIC_DISPUTE_RESOLVER_ADDRESS`     | Yes           | DisputeResolver contract address              |

## Platform Server (Webhook + Backend Calls)

These are server-side only variables — never expose them to client bundles:

| Variable                   | Required | Description                                                        |
| -------------------------- | -------- | ------------------------------------------------------------------ |
| `FLOWSTATE_API_KEY`        | Yes      | Server-side API key for direct backend calls                       |
| `FLOWSTATE_WEBHOOK_SECRET` | Yes      | Secret returned by `/webhooks/register` for signature verification |
| `FLOWSTATE_API_BASE_URL`   | No       | Override the default `https://api.flowstate.xyz`                   |

<Callout type="warning">
  Never expose `FLOWSTATE_API_KEY` or `FLOWSTATE_WEBHOOK_SECRET` in client-side
  code. These should only be used in server-side API routes or serverless
  functions.
</Callout>

## Optional: Self-Hosted Operator Variables

If your team runs the FlowState backend infrastructure directly, the backend service requires these variables (defined in `backend/src/config/env.ts`):

| Variable                  | Required | Default | Description                                   |
| ------------------------- | -------- | ------- | --------------------------------------------- |
| `DATABASE_URL`            | Yes      | —       | PostgreSQL connection string                  |
| `SHIPPO_KEY`              | Yes      | —       | Shippo API test key for shipping integration  |
| `PORT`                    | No       | `3001`  | Backend server port                           |
| `REDIS_URL`               | No       | —       | Redis connection URL; enables BullMQ queues   |
| `PLATFORM_FEE_BPS`        | No       | `250`   | Platform fee in basis points (2.5%)           |
| `GRACE_PERIOD_HOURS`      | No       | `48`    | Hours after delivery before auto-finalization |
| `PINATA_BUYER_AGENT_URL`  | No       | —       | Pinata AI agent URL for buyer chat            |
| `PINATA_SELLER_AGENT_URL` | No       | —       | Pinata AI agent URL for seller chat           |
| `PINATA_ADMIN_AGENT_URL`  | No       | —       | Pinata AI agent URL for admin chat            |

See [Internal Ops (Legacy)](/self-hosting/local-dev) for full local development setup.
